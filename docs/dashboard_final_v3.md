# Dashboard Final v3 Summary

## 1. 인증 및 GNB 강화 ✅ 완료

- **AS-IS**: 로그인은 각 페이지 컴포넌트 내부 상태로 판단했고, GNB는 단일 메뉴로 고정되어 있습니다. 이동 시 토큰을 다시 확인하거나 실패한 경우가 있었습니다.
- **TO-BE**: JWT를 `AuthContext` 또는 글로벌 상태(예: `AppProvider`)로 보관해 페이지 이동/새로고침에도 그대로 유지하고, 로그인하지 않은 상태에서는 대시보드 관련 라우트를 차단합니다. GNB는 메인과 서브(대시보드) 영역에서 각각 다른 메뉴를 노출하며, 로그인 여부에 따라 “로그인 → 대시보드”로 텍스트를 바꿉니다.

## 2. 대시보드 상단 및 블로그 등록 ✅ 완료

- **AS-IS**: 대시보드 헤더는 고정 텍스트 “AI Blog Auto-Pilot”이며, 블로그 등록 UI는 단일 폼으로 되어 있었습니다.
- **TO-BE**: 로그인한 사용자의 이름을 반영해 “[사용자명]의 AI 대시보드” 또는 미로그인 시 “고객님의 AI 대시보드”로 타이틀을 바꿉니다. 블로그 카드는 등록된 블로그만 보여주고, 남은 슬롯은 “+ 등록” 버튼으로 표시합니다. 등록 가능한 플랫폼은 네이버, Blogger, 티스토리, 인블로그, 워드프레스까지 확장하며, 각 카드/등록 폼에 URL과 API 키(또는 인증 토큰) 입력란을 포함합니다.

## 3. 블로그별 상세 설정 ✅ 완료

- **AS-IS**: 입력창 대부분 영문 placeholder, 페르소나 셀렉트, 분석 버튼이 즉시 호출되며 실시간 preview와 HTML 생성 결과를 별도로 관리했습니다.
- **TO-BE**: 모든 입력창을 한글 레이블/placeholder로 구성하고, 페르소나는 셀렉트 대신 자유 입력(예: “전문 SEO 마케터처럼”). “분석하기” 버튼은 최초 한 번만 Gemini 호출을 트리거하며 결과가 생성되면 버튼 텍스트를 “다시 분석하기”로 바꿔 판단하도록 합니다. `Live Preview`와 `HTML 생성` 결과를 [포스팅 현황] 메뉴에 기록하고, 목록에서 언제든 조회할 수 있게 백엔드 저장/조회 API를 연동합니다.

## 4. 포스팅 현황 및 키워드 트래킹 ✅ 완료

-- **TO-BE**: 포스팅 리스트 컬럼을 "발행 글 제목 | 날짜 | 키워드 | 플랫폼 | 순위 | 변동 | 업데이트 | 원문링크" 순으로 재구성하고, 각 행에서 상세 트래킹 페이지(세부 데이터)를 열 수 있도록 내비게이션을 추가합니다. 키워드/순위/변동 데이터는 백엔드의 실제 `posts/keywords` 응답과 연결하며, 각 포스팅 항목에서 `/status` 상세 링크를 통해 더 많은 값으로 이동할 수 있게 했습니다.

## 5. 데이터 연동 및 기타 ✅ 완료

-- **TO-BE**: 크레딧 잔여량, 일정, 키워드, 이미지 상태 등 가능한 모든 상태를 실제 API로부터 받아 표시하며, `useMemo` 기반 실시간 크레딧 계산 로직(기본 5 + 이미지 1장당 1 + 500자 초과 시 1 크레딧)과 이미지 경로 최적화(완료된 이미지부터 렌더)도 계속 활용합니다. 목록에 보여줄 데이터는 백엔드가 제공하는 리얼 응답을 기준으로 하고, 미개발(크레딧 충전 등)의 경우 안내 메시지로 대체해 사용자 불편을 줄입니다. 이 계산식은 백엔드 `calculate_required_credits`를 통해 검증/변경 가능하며, 프론트는 `preview` 호출 시 받은 `credits_required` 값을 즉시 알림 영역에 보여줍니다.

## Next steps

1. 프론트 전역 `AuthContext`를 구현해 `/app` 수준의 라우터 가드와 조합합니다.  
2. `backend`의 `blogs/analyze`·`posts/preview`·`posts/status` API를 위 문서 그대로 배포하고, 대시보드에서 새롭게 필요한 필드를 (`credits_required`, `summary`, `tracking_url`)을 반환하도록 정리합니다.  
3. 대시보드에서 생긴 새로운 입력(한글화된 프롬프트, word range)을 기존 `PreviewRequest` 타입에 반영해 타입 안정성을 확보합니다.
